Conceitos

  • Software as a Service (PaaS)
  • Infrastructure as a Service (IaaS)
  • Platform as a Service PaaS

Conceitos - SaaS

  • É um modo de disponibilizar uma aplicação na internet como um serviço;
  • Não há necessidade de instalar uma copia para cada cliente;
  • Livra o seu software de complexidades de hardware e de sistemas operacionais encontrados nos clientes;
  • Quem mantém o SaaS deve se preocupar com o acesso, segurança, disponibilidade e desempenho;

Conceitos - SaaS

  • O cliente só paga pelo serviço (ou não), não é necessário pagar por hardwares, instalações, manutenções e atualizações;
  • Exemplos: BRbid.com, BRpatio.com, sites bancários, sites de compras online e etc

Conceitos - IaaS

  • Disponibilização recursos computacionais virtualizados;
  • Não ha necessidade dos clientes se preocuparem com falhas no hardware, upgrade, quedas de energia e etc;
  • Provê aplicações para controlar a infraestrutura, gerenciar tarefas e fazer a manutençãoda mesma;

Conceitos - IaaS

  • O valor pago pela infraestrutura é relacioado com os recursos contratados, tipicamente cobrado em horas/máquina;
  • Como a infraestrutura é tercerizada, o monitoramento pode ser mais dificil se não houver ferramentas para isso;
  • Exemplos: Amazon Web Services (AWS), Windows Azure, Google Compute Engine (GAE), Digital Ocean e etc

Conceitos - PaaS

  • É um modelo para executar aplicações sem a preocupação de manter o hardware e a infraestrutura;
  • O provedor entrega ferramentas de hardware e de software, geralmente necessárias para o desenvolvimento de aplicativos;
